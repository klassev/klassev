# Тестирование доступа к админ-панели

## Что нужно сделать

Я проверяю, что админ-панель Filament доступна, работает корректно, вход выполняется успешно и все базовые функции работают.

## Почему это важно

Тестирование доступа необходимо потому что:
- Убеждаюсь, что админ-панель настроена правильно
- Проверяю безопасность доступа
- Выявляю возможные проблемы на раннем этапе
- Убеждаюсь, что все компоненты работают
- Подтверждаю готовность к дальнейшей разработке

## Пошаговая инструкция

1. Проверяю доступность админ-панели:
   Открываю в браузере: https://klassev.test/admin
   Должна отобразиться страница входа в админ-панель.

2. Проверяю страницу входа:
   Убеждаюсь, что:
   - Форма входа отображается
   - Поля email и password присутствуют
   - Кнопка входа работает
   - Нет ошибок на странице

3. Выполняю вход с админ-учетными данными:
   Ввожу:
   - Email: admin@klassev.test
   - Password: admin123
   Нажимаю "Войти" или "Login".

4. Проверяю успешный вход:
   После входа должна отобразиться главная страница админ-панели (Dashboard).

5. Проверяю навигацию:
   Убеждаюсь, что:
   - Боковое меню отображается
   - Навигационные группы видны
   - Логотип и брендинг отображаются правильно

6. Проверяю Dashboard:
   На главной странице проверяю:
   - Нет ошибок
   - Статистика отображается (если настроена)
   - Виджеты работают (если есть)

7. Проверяю выход из системы:
   Нажимаю на аватар/имя пользователя в правом верхнем углу.
   Выбираю "Выйти" или "Logout".
   Должен произойти выход и возврат на страницу входа.

8. Проверяю защиту роутов:
   Пытаюсь открыть https://klassev.test/admin напрямую без авторизации.
   Должен произойти редирект на страницу входа.

9. Проверяю через curl (опционально):
   ```bash
   curl -I https://klassev.test/admin
   ```
   Должен вернуться HTTP код (302 редирект или 200, если авторизован).

10. Проверяю логи на наличие ошибок:
    ```bash
    tail -f storage/logs/laravel.log
    ```
    Открываю админ-панель и проверяю, что нет ошибок в логах.

11. Проверяю консоль браузера:
    Открываю Developer Tools (F12) в браузере.
    Проверяю вкладку Console на наличие JavaScript ошибок.

12. Проверяю Network запросы:
    В Developer Tools проверяю вкладку Network.
    Убеждаюсь, что все запросы выполняются успешно (статус 200 или 302).

13. Тестирую на разных браузерах (опционально):
    Проверяю работу в:
    - Chrome/Edge
    - Firefox
    - Safari (если на macOS)

14. Проверяю адаптивность (опционально):
    Открываю Developer Tools и переключаюсь в режим мобильного устройства.
    Проверяю, что админ-панель корректно отображается на мобильных.

15. Проверяю темную тему (если включена):
    Переключаю тему (если есть переключатель).
    Убеждаюсь, что темная тема работает корректно.

## Что проверить после выполнения

- [ ] Админ-панель доступна по /admin
- [ ] Страница входа отображается
- [ ] Вход выполняется успешно
- [ ] Dashboard загружается без ошибок
- [ ] Навигация работает
- [ ] Выход из системы работает
- [ ] Защита роутов работает (неавторизованный доступ блокируется)
- [ ] Нет ошибок в логах
- [ ] Нет ошибок в консоли браузера

## Возможные проблемы

- Если админ-панель не открывается, проверяю роуты: `php artisan route:list | grep admin`
- Если ошибка 404, проверяю что Filament установлен и настроен
- Если ошибка 500, проверю логи: `tail -f storage/logs/laravel.log`
- Если не могу войти, проверяю что пользователь создан и пароль правильный
- Если ошибки JavaScript, проверяю что assets скомпилированы

## Дополнительные проверки

- Проверяю производительность загрузки страниц
- Проверяю работу фильтров и поиска (когда будут ресурсы)
- Проверяю работу форм (когда будут созданы)
- Проверяю работу таблиц (когда будут созданы)

## Следующие шаги

После успешного тестирования админ-панели можно:
- Создавать Filament Resources для моделей
- Настраивать формы и таблицы
- Добавлять фильтры и поиск
- Настраивать дополнительные функции
