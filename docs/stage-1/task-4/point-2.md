# Настройка админ-панели (php artisan filament:install)

## Что нужно сделать

Я выполняю команду установки Filament, которая создаст необходимые файлы конфигурации, опубликует ресурсы и настроит базовую структуру админ-панели.

## Почему это важно

Команда `filament:install` необходима потому что:
- Создает структуру админ-панели
- Публикует конфигурационные файлы
- Настраивает провайдеры и сервисы
- Создает необходимые директории
- Настраивает роутинг для админ-панели
- Подготавливает базу для дальнейшей настройки

## Пошаговая инструкция

1. Выполняю команду установки Filament:
   ```bash
   php artisan filament:install
   ```

2. Отвечаю на вопросы установщика:
   Установщик может спросить:
   - Какой провайдер использовать (выбираю `panels`)
   - Имя панели (использую `admin` или оставляю по умолчанию)
   - Путь к панели (обычно `/admin`)

3. Ожидаю завершения установки:
   Команда создаст необходимые файлы и структуру.

4. Проверяю созданные файлы:
   ```bash
   ls -la app/Providers/
   ```
   Должен быть создан `FilamentServiceProvider` или подобный файл.

5. Проверяю конфигурацию:
   ```bash
   ls -la config/
   ```
   Должен быть файл `filament.php` или подобный.

6. Проверяю опубликованные ресурсы:
   ```bash
   ls -la resources/views/
   ```
   Могут быть созданы view файлы для Filament.

7. Проверяю роуты:
   ```bash
   php artisan route:list | grep admin
   ```
   Должны быть созданы роуты для админ-панели.

8. Очищаю кэш конфигурации:
   ```bash
   php artisan config:clear
   php artisan route:clear
   php artisan view:clear
   ```

9. Проверяю доступность админ-панели:
   Открываю в браузере: https://klassev.test/admin
   Должна отобразиться страница входа в админ-панель (или ошибка, если еще не создан пользователь).

10. Проверяю логи на наличие ошибок:
    ```bash
    tail -f storage/logs/laravel.log
    ```
    Убеждаюсь, что нет критических ошибок.

## Что проверить после выполнения

- [ ] Команда установки выполнена без ошибок
- [ ] Провайдер Filament создан
- [ ] Конфигурационный файл создан
- [ ] Роуты админ-панели зарегистрированы
- [ ] Админ-панель доступна по /admin
- [ ] Нет ошибок в логах

## Структура после установки

После установки должны быть созданы:
- Провайдер для Filament (app/Providers/)
- Конфигурационный файл (config/filament.php)
- Возможно view файлы (resources/views/)
- Роуты для админ-панели

## Возможные проблемы

- Если команда не найдена, проверяю что Filament установлен: `composer show filament/filament`
- Если ошибка с правами доступа, проверяю права на папки storage и bootstrap/cache
- Если админ-панель не открывается, проверяю роуты: `php artisan route:list`
