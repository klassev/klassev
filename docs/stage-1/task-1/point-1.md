# Выбор и установка фреймворка (Laravel)

## Что нужно сделать

Я устанавливаю Laravel 12 как основной фреймворк для проекта.

## Почему Laravel 12

Laravel 12 — актуальная версия (выпущена в феврале 2025 года), которая будет поддерживаться до 2028 года. Это важно для долгосрочной поддержки проекта. Laravel выбран потому что:
- Это самый популярный PHP фреймворк, что релевантно для собеседований по PHP
- Быстрая разработка благодаря встроенным возможностям
- Отличная документация и большое сообщество
- Встроенная поддержка аутентификации, миграций, очередей

## Пошаговая инструкция

1. Проверяю, что у меня установлен PHP 8.4 или выше:
   ```bash
   php -v
   ```
   Если версия ниже 8.4, обновляю PHP.

2. Проверяю наличие Composer:
   ```bash
   composer --version
   ```
   Если Composer не установлен, устанавливаю его согласно официальной документации.

3. Создаю новый Laravel проект:
   ```bash
   composer create-project laravel/laravel . --prefer-dist
   ```
   Использую точку (.) чтобы создать проект в текущей директории, так как я уже нахожусь в папке проекта.

4. Проверяю, что установка прошла успешно:
   ```bash
   php artisan --version
   ```
   Должна отобразиться версия Laravel 12.x.

5. Проверяю структуру проекта:
   ```bash
   ls -la
   ```
   Должны быть созданы стандартные папки Laravel: app, config, database, routes, и т.д.

## Что проверить после выполнения

- [x] Laravel установлен и работает (`php artisan --version`)
- [x] Структура проекта создана корректно
- [x] Нет ошибок при установке

## Возможные проблемы

- Если возникает ошибка с правами доступа, проверяю права на папку проекта
- Если Composer не может найти пакет, проверяю подключение к интернету и настройки Composer
