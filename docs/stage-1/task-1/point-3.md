# Настройка Git репозитория

## Что нужно сделать

Я инициализирую Git репозиторий, настраиваю .gitignore и делаю первый коммит с базовой структурой проекта.

## Почему это важно

Версионный контроль необходим для:
- Отслеживания изменений в коде
- Возможности отката к предыдущим версиям
- Совместной работы (если проект будет разрабатываться командой)
- Резервного копирования кода
- Интеграции с CI/CD системами

## Пошаговая инструкция

1. Проверяю, установлен ли Git:
   ```bash
   git --version
   ```
   Если Git не установлен, устанавливаю его.

2. Проверяю, не инициализирован ли уже репозиторий:
   ```bash
   ls -la .git
   ```
   Если папка .git существует, репозиторий уже инициализирован. Если нет, продолжаю.

3. Инициализирую Git репозиторий:
   ```bash
   git init
   ```

4. Настраиваю имя пользователя и email (если еще не настроено глобально):
   ```bash
   git config user.name "Мое Имя"
   git config user.email "myemail@example.com"
   ```
   Или использую глобальные настройки:
   ```bash
   git config --global user.name "Мое Имя"
   git config --global user.email "myemail@example.com"
   ```

5. Проверяю содержимое .gitignore:
   ```bash
   cat .gitignore
   ```
   Убеждаюсь, что там есть:
   - /vendor
   - /node_modules
   - .env
   - /.idea
   - /.vscode
   - и другие служебные файлы

6. Добавляю все файлы в staging area:
   ```bash
   git add .
   ```

7. Проверяю, что будет закоммичено:
   ```bash
   git status
   ```
   Убеждаюсь, что .env и другие чувствительные файлы не попадут в репозиторий.

8. Делаю первый коммит:
   ```bash
   git commit -m "Initial commit: Laravel 12 project setup"
   ```

9. (Опционально) Если у меня есть удаленный репозиторий (GitHub, GitLab и т.д.), добавляю его:
   ```bash
   git remote add origin https://github.com/username/repository.git
   git branch -M main
   git push -u origin main
   ```

## Что проверить после выполнения

- [ ] Git репозиторий инициализирован (папка .git существует)
- [ ] Первый коммит создан (`git log` показывает коммит)
- [ ] .env файл не попал в репозиторий (проверяю через `git status`)
- [ ] vendor и node_modules игнорируются

## Важные замечания

- Никогда не коммичу .env файл с реальными ключами и паролями
- Регулярно делаю коммиты с понятными сообщениями
- Использую ветки для разработки новых функций (git flow)
