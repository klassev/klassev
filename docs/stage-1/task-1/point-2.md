# Настройка структуры проекта

## Что нужно сделать

Я настраиваю базовую структуру проекта, создаю необходимые директории и проверяю организацию файлов согласно стандартам Laravel.

## Почему это важно

Правильная структура проекта обеспечивает:
- Легкость навигации по коду
- Соответствие стандартам Laravel (PSR-4)
- Удобство для других разработчиков
- Возможность масштабирования проекта

## Пошаговая инструкция

1. Проверяю текущую структуру проекта:
   ```bash
   tree -L 2 -I 'vendor|node_modules'
   ```
   Или использую `ls -la` для просмотра основных директорий.

2. Создаю дополнительные директории, которые понадобятся для проекта (Laravel уже создал базовую структуру, добавляю только кастомные):
   ```bash
   mkdir -p app/Services
   mkdir -p app/Repositories
   ```
   Остальные директории (app/Http/Requests, app/Http/Middleware, app/Models, database/seeders, database/factories, tests/Feature, tests/Unit) уже созданы Laravel автоматически при установке.

3. Проверяю структуру app/Http/Controllers:
   ```bash
   ls -la app/Http/Controllers
   ```
   Должен быть базовый Controller.php (создан Laravel автоматически).

4. Создаю базовую структуру для API (это кастомная структура для версионирования):
   ```bash
   mkdir -p app/Http/Controllers/Api/V1
   ```
   Это нужно для версионирования API (v1, v2 и т.д.). Laravel не создает эту структуру автоматически, так как это специфично для проекта.

5. Проверяю наличие файла .gitignore:
   ```bash
   cat .gitignore
   ```
   Убеждаюсь, что vendor, node_modules, .env и другие служебные файлы игнорируются.

6. Создаю README.md с описанием проекта (если его еще нет):
   ```bash
   touch README.md
   ```

## Что проверить после выполнения

- [x] Все необходимые директории созданы
- [x] Структура соответствует стандартам Laravel
- [x] .gitignore настроен корректно
- [x] README.md создан (можно заполнить позже)

## Дополнительные заметки

Структура проекта будет расширяться по мере разработки, но базовая организация должна быть заложена с самого начала. Это упростит дальнейшую работу и поддержку кода.
